selfHelp 장르이지만 단계형 프레임워크 없이 핵심 주장을 다각도로 논증하는 책은 argument 구조로 분류한다.

**대표 사례: 린치핀 (Seth Godin)**

- 핵심 패턴: "기존 방식(X)은 더 이상 통하지 않는다. Y가 되어야 한다"
- hookStrategy: contrarian (통념 반박이 책의 전체 동력)
- narrativeArcType: warning (지금 이대로 가면 위험하다는 경고)
- emotionalTone: provocative + uplifting (도발 후 고무)
- urgencyLevel: high (지금 선택해야 한다)
- contentMode: mixed (개념 중심이지만 실행 방향도 제시)

**framework 구조 selfHelp와의 차이:**

- framework: 단계가 있고 "이렇게 하면 된다" → instruction/transformation
- argument: 단계 없이 "왜 그래야 하는가"를 논증 → warning/discovery
- framework 책의 coreFramework는 단계 이름을 나열, argument 책은 핵심 역량/자질 3요소 형태로 표현

**How to apply:** 책 요약에서 "단계별 실천법"보다 "개념의 대비"와 "통념 반박"이 지배적이면 argument로 분류한다. keyConceptCount는 저자가 제시하는 핵심 자질/요소 수(보통 3)와 일치시킨다.
