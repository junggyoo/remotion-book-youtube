selfHelp + framework 구조 책은 두 가지 하위 유형으로 나뉜다.

**유형 A: 루틴/습관 중심 (예: 미라클 모닝)**

- hookStrategy: transformation (저자의 개인 서사가 핵심)
- narrativeArcType: transformation
- emotionalTone: uplifting + disciplined
- urgencyLevel: high (지금 당장 시작하라)
- uniqueElements: 루틴 시각화, 타임라인

**유형 B: 시스템/원리 중심 (예: Atomic Habits)**

- hookStrategy: contrarian (통념 반박이 핵심)
- narrativeArcType: instruction
- emotionalTone: disciplined + hopeful
- urgencyLevel: medium (꾸준히 하면 된다)
- uniqueElements: 수식/데이터 시각화, 순환 다이어그램, 정체성 레이어

**How to apply:** 책 요약에서 저자 개인 경험 비중이 높으면 유형 A, 수학적 논거/시스템 설계 비중이 높으면 유형 B로 분류한다. keyConceptCount는 프레임워크 단계 수와 일치시킨다 (SAVERS=6, Four Laws=4).
